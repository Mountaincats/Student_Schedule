关于当前代码的一些介绍：
使用Android Studio开发，模板是Empty Activity。
目前已经实现了Schedule, Day, Week的数据模型，以及RepeatRule工具类


关于项目开发思路的一些介绍：

结构上，以Schedule为基本单元，Day为次级基本单元，Week作为最终单元来管理基本单元的行为，并且提供一些必要的数据记录，比如说：记录本周是第几周，以及单双周，以及每周一的日期（Week是从周一到周日）。

具体来说，用户创建的Schedule会依附于对应的Day，而Week则是控制许多个Day的行为，比如：决定某一天的重复，是每周重复一次，还是单双周重复一次。

为了方便重复性行为的控制，额外创建了RepeatRule工具类。

流程上，目前我的理解是，开发一个功能大致分为三件事：
1.完成数据模型的创建
2.实现不同模型之间的交互逻辑
3.设置数据存储
4.设计界面布局和用户交互

具体的某个流程，或许可以询问Ai，查询相关流程实现在Android Studio是如何做到的
